Analytical Approaches & Justification

The identified analytical approaches selected for the aerospace and defense industry are: Malware Analysis, Event Correlation, and Anomaly Detection. These approaches were selected because they will help organizations within this industry make timely, relevant, and effective decisions considering their cybersecurity posture.

Analytical Approaches

  • Malware analysis was chosen because malware is one of the top threat trends identified within the industry. The aerospace & defense industry’s organizations and individuals are very susceptible to attackers dropping malicious software or files to exploit their network.

  • Value added in creating timely, relevant, and actionable CTI:
    • It also dissects malware to understand how it works, how best to identify it, how to eliminate it, and how to prevent it from occurring again.
    • It allows organizations to extract important information and details to enhance their CTI.
    • It can provide detailed information needed to respond to network intrusions (which can explain what happened, where the machines and files are located, and it can also find the signatures for the intrusion detection systems)

  • Data sources required: malware, capability to view code without compromising systems

  • Tools used for analytics: FireEye

  • Key steps to malware analysis are effective reconstruction of the exploit and correctly identifying the malware used.
    • The shells and code design are used to understand the threat and the target of the exploit.
    • Other alternatives to discovering the attacker are to attempt to track the origins of the exploit or to wait for somebody to claim the attack.
  • Event Correlation was selected because it analyzes the relationships between events and activities. Within the A&D industry, this is helpful in analyzing threats, such as malware attacks and data breaches.

  • Value added in creating timely, relevant, and actionable CTI:
    • It can gain deeper insights into collected data by integrating multiple data sources together.
    • Organizations can also customize rules and alerts to get the most value out of this analytical approach.

  • Data sources required: server logs/ SIEM

  • Tools used for analytics: AlienVault and FireEye

  • Key steps are understanding the portions of a cyberattack that are recurring and knowing which events are important (ie signal vs. noise).
    • Evaluation measures have to be taken to understand fidelity of assessments that events, files, etc. are in fact linked.
    • Other approaches for this task are closely related to anomaly detection but understanding that indications and warnings of an impending attack are critical to heed.
  • Anomaly detection was selected because it can detect malicious activity and provide critical and actionable intelligence. Within the A&D industry, this would be especially useful when trying to detect attacks, such as data breaches, malware attacks, and supply chain attacks.

  • Value added in creating timely, relevant, and actionable CTI:
    • It can identify any event falling outside of a predefined set of normal behaviors and detect malicious activity.
    • Can be a key indicator for espionage activities from insiders with privileged accesses.

  • Data sources required: Access logs/SIEM, network monitoring

  • Tools used for analytics:
    • Rules set at the system administration level to better understand normal work hours and when a log on is suspicious.
    • Notifications when a user is accessing data outside of the scope of their work
    • Abnormal files and types, encryptions, etc. within logical structure

  • Key steps are ensuring that a baseline has been established and maintained is vital, because with no baseline, there can be no deviations from a “normal”.
    • Social engineering can help discover coworkers that are complicit in some cases
    • Alternatives could be using machine learning to understand normal patterns within the system and to stop odd behavior and alert system administrators
